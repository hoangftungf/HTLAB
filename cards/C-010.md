# C-010 - Value, boolean, and control-flow foundation

status: done
deps: C-009
implements: FR3, FR7 (generator and interpreter foundation)

## Scope

Build the expression and control-flow foundation needed before adding the rest of Motion, Patrol line, Sensor, and side-effect blocks. This card should make Blockly reporters and booleans first-class in IR, rather than forcing everything through `args: number[]`.

1. **Value expressions** - support numeric literals, variables, sensor reads, math operators, unary math functions, random, remainder, and trig with explicit `degree`/`radian` mode.
2. **Boolean expressions** - support comparisons, not-equal, `and`, `or`, `not`, sensor booleans, and remote-control stub boolean behavior.
3. **Control flow** - implement `if`, `if else`, `repeat N`, `repeat forever`, `repeat until`, `wait`, `wait until`, `break`, and `return` semantics where they are safe in the simulator.
4. **Runtime guards** - enforce instruction caps, loop caps, `wait until` timeout, invalid math domain diagnostics, and deterministic random behavior from the existing seeded RNG.
5. **Generator shape** - refactor generator handlers so a Blockly block can produce a command sequence, value expression, boolean expression, or diagnostic.
6. **Tests** - add interpreter and generator tests for nested expressions and loop/branch behavior.
