# C-004 — Telemetry recorder + deterministic replay

status: done
deps: C-001, C-002, C-003
implements: FR6

## Scope

Build the telemetry recording and replay system that captures every tick's full state and can replay it exactly.

1. **Telemetry recorder** — after each `tick()`, capture TelemetryFrame { tick, robot, sensors, motorTargets } into an array buffer
2. **Telemetry export** — `sim.getTelemetry(): TelemetryFrame[]` returns full array for saving/display
3. **Replay mode** — `createReplay(telemetry: TelemetryFrame[]): Simulation` — creates a read-only sim that replays frames tick by tick (no physics, just playback)
4. **Replay controls** — replay as regular sim with run/pause/reset/speed, step forward/backward (by frame)
5. **Diff tool** — compare 2 telemetry logs side by side: which tick did trajectories diverge?
6. **Memory management** — configurable ring buffer (default keep last 3600 frames = 60 seconds), auto-truncate
