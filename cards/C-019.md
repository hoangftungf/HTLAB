## Scope

4 bug fixes for My Blocks:

1. **Remove `my_block_param_value` from flyout** — switch My Blocks toolbox category to a fully dynamic flyout callback (`MY_BLOCKS_FLYOUT`) so only "Create new blocks" button + generated call blocks appear.
2. **Block-type selector working** — add `blockType: "statement"|"reporter"|"boolean"` to dialog state; 3 toolbar buttons toggle the type; define block stores `blockType_` in mutator; call block in flyout uses correct connection type (prev/next vs output Number vs output Boolean).
3. **Call block appears in flyout after OK** — dynamic flyout callback reads all `my_block_definition` blocks on the workspace and generates call-block XML; `workspace.refreshToolboxSelection()` called after create/edit.
4. **NAME field read-only** — change `FieldTextInput` → `FieldLabel` in `my_block_definition._rebuildInputs` and `_callMixin.init` so the name cannot be edited directly in the block; only the Edit function modal can change it.
