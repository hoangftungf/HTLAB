# C-012 - Sensor, Light Speaker, and AI compatibility blocks

status: done
deps: C-011
implements: FR4, FR5, FR7 (sensor and side-effect compatibility)

## Scope

Add the non-motion block categories needed for toolbox/workspace parity while keeping runtime behavior honest. Blocks backed by simulator data should work; blocks without simulated hardware should produce telemetry/diagnostics rather than silently doing nothing.

1. **Sensor reporters and booleans** - implement integrated grayscale, single grayscale where mappable, ultrasonic placeholder distance, ambient light, temperature, humidity, flame, magnetic, volume, motor encoder, timer, color sensor, and touch/infrared booleans according to available simulator state.
2. **Timer and encoder state** - add runtime state for current timer, reset timer, motor encoder read, and reset motor encoder if not already available.
3. **Remote control stub** - implement `remote control button` as a documented boolean stub returning `false` with diagnostic/comment.
4. **Light Speaker side effects** - add telemetry-only commands for sound, LED RGB, LED off, emotion screen, digital tube, screen display, clear screen, clear digital tube, electromagnet, and `reading 1` stub.
5. **AI blocks** - include documented AI blocks in registry/toolbox with `stub` or `telemetry-only` status unless a real model/runtime exists.
6. **Telemetry events** - extend telemetry so side-effect blocks can be inspected without changing robot kinematics.
7. **Warnings** - unsupported hardware blocks must be visible in generated diagnostics and, where useful, the UI status panel.
