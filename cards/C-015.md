# C-015 - Full toolbox parity, QA, and integration polish

status: done
deps: C-014
implements: FR3, FR7, FR8 (full block integration QA)

## Scope

Finish the expanded block integration by making the UI, toolbox, generator, save/load, diagnostics, and sample programs coherent as one user-facing workflow. This card is the acceptance pass for all documented WhalesBot blocks.

1. **Toolbox parity** - every documented category and block appears in the Blockly toolbox with correct grouping, labels, field defaults, dropdowns, and output/statement shape.
2. **Workspace compatibility** - users can drag, connect, save, reload, and regenerate IR from representative blocks across every category.
3. **Diagnostics UX** - unsupported, telemetry-only, stub, and blocked-by-sandbox blocks show clear diagnostics in the app without breaking the rest of the program.
4. **Sample programs** - add or update sample programs that cover: line-following route, math/logic/control-flow, sensors, side-effect telemetry, variables/custom block, and C Code disabled/sandbox behavior.
5. **Regression suite** - add smoke coverage for Blockly -> IR -> Interpreter -> Telemetry -> Save/Load.
6. **Docs alignment** - update user-facing docs so every category has an implementation status and known limitation.
7. **Visual QA** - verify the full app layout remains usable with the larger toolbox on desktop and mobile-sized viewports.
