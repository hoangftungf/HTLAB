# C-002 — Grayscale 5-in-1 sensor model

status: done
deps: C-001
implements: FR2, FR4

## Scope

Implement the grayscale 5-in-1 line sensor model inside SimulationCore. This is the hardest feature in MVP — a C-grade feature per scope.

1. **Pixel sampling** — read 5 sensor positions from map ImageData each tick, based on robot position + heading + sensor geometry (row of 5 sensors at front of robot, configurable spacing)
2. **Road value 0-100** — each of 5 roads returns 0 (white) to 100 (black), computed from pixel grayscale + threshold
3. **Pattern detection** — binary on/off per road → pattern string like `00100`, `01110`, `11100`
4. **Line position** — weighted position -100..+100 (far left to far right), 0 = line centered, usable for P-controller
5. **Sensor groups** — left (Road 1-2-3), middle (Road 2-3-4), right (Road 3-4-5), each returns detected bool
6. **Calibration** — `CALIBRATE_GRAYSCALE`: sample white floor + black line, compute per-road thresholds
7. **Configurable noise** — sensorNoise (gaussian stddev), sensorBias (offset), latency (tick delay)
