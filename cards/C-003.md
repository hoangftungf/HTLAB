# C-003 — IR Interpreter: domain-specific instruction set

status: done
deps: C-001, C-002
implements: FR3

## Scope

Build the IR (Intermediate Representation) interpreter that executes robot programs step-by-step each simulation tick. Blockly generates IR → Interpreter executes IR → calls sim.setMotors() + reads sim.state.sensors.

1. **IRProgram + IRCommand types** — TypeScript types matching the contract in `05-contract.md`
2. **Interpreter** — `createInterpreter(program, sim)` returns interpreter object with `step(): boolean`, `reset()`, `done: boolean`
3. **All 14 opcodes** — INIT_HARDWARE, CALIBRATE_GRAYSCALE, SET_MOTOR, WAIT_TICKS, READ_SENSOR_ROAD, READ_SENSOR_GROUP, READ_LINE_POSITION, IF_SENSOR_VALUE, SET_VAR, LABEL, JUMP, LOOP_START, LOOP_END, END_PROGRAM
4. **Execution limits** — max 1000 instructions per tick, runtime cap (configurable, default 5 minutes = 18000 ticks)
5. **ACC register + variables** — single accumulator register + 8 variable slots (0-7) for SET_VAR/GET_VAR
6. **Wait handling** — WAIT_TICKS pauses instruction pointer, motors keep running, resumes after N ticks
