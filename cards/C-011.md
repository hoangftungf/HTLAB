# C-011 - Full Motion and Patrol line runtime

status: todo
deps: C-010
implements: FR2, FR3, FR7 (robot behavior blocks)

## Scope

Implement the blocks that directly affect robot movement and line-following behavior. This card should make the simulator capable of running the documented Motion and Patrol line command set, with clear diagnostics for hardware modes that remain approximations.

1. **Motion blocks** - implement two-motor movement, timed motor movement, individual motor power, stop motor, reverse motor, all-stop, encoder/angle-based movement where supported, and safe stubs for steering/omni-wheel if the differential-drive simulator cannot represent them yet.
2. **Patrol line initialize** - implement both differential-drive and omni-wheel initialization blocks as configuration commands; omni-wheel may be a diagnostic/stub until the physics model supports it.
3. **Black and white detection** - keep calibration behavior compatible with the current grayscale model and document threshold assumptions.
4. **Patrol line commands** - implement continuous patrol, timed patrol, patrol until intersection, rush-through timing, and turn by left/middle/right sensor group.
5. **Start motor variants** - support timed, angle/encoder, and sensor-condition termination variants where the simulator has data; otherwise emit explicit diagnostics.
6. **Start button stub** - keep `start button` in the registry/toolbox as an intentional no-op diagnostic.
7. **Safety** - add max-turn/max-patrol timeout errors so a bad track or missing line does not run forever.
