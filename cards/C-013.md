# C-013 - Variables and My Blocks execution model

status: todo
deps: C-012
implements: FR3, FR7 (variables and custom blocks)

## Scope

Implement the variable and custom-block model needed for larger WhalesBot programs. This card should cover symbol tables, variable mutation, procedure definition/call, parameters, and return behavior in a deterministic interpreter model.

1. **Variable registry** - support create/get/set/change variable operations with stable ids and names persisted in Blockly XML and generated IR.
2. **Expression integration** - variables can be used in math, logic, sensor thresholds, wait conditions, loop counts, and motor power fields.
3. **My Blocks definitions** - support custom block declarations with typed parameters where Blockly exposes them.
4. **Call frames** - implement a simple call stack with local parameter values, return value handling, recursion guard, and max-call-depth guard.
5. **Return semantics** - wire `Return [value]` to custom-block calls; outside a callable context it should emit a diagnostic instead of crashing.
6. **Save/load compatibility** - project save/load must preserve variables, custom blocks, and generated IR.
7. **Tests** - cover variable mutation, nested custom-block calls, return value usage, and recursion/depth diagnostics.
