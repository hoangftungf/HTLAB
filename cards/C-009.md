# C-009 - Block registry + IR v2 contract

status: done
deps: C-008
implements: FR3, FR7 (expanded block coverage)

## Scope

Create the contract and registry foundation for integrating every block documented in `docs/whalesbot-block-studio-dac-ta-block.md`. This card defines shapes and metadata only; it should not attempt full runtime behavior for every block.

1. **Contract update** - amend `flow/05-contract.md` with an IR v2 section that can represent commands, value expressions, boolean expressions, diagnostics, metadata, text/color enum fields, and C-code payloads.
2. **Block registry** - create a canonical registry for all documented WhalesBot categories: Motion, Light Speaker, Sensor, Event, Loop, Logic, Math, Variable, AI, Patrol line, My Blocks, C Code.
3. **Runtime status model** - every registry entry must declare one of: `implemented`, `telemetry-only`, `stub`, `blocked-by-sandbox`.
4. **Field schema** - every block entry must declare its Blockly type id, category, statement/output kind, fields, defaults, dropdown values, and generator/runtime handler id.
5. **Compatibility decisions** - encode the decisions from doc section 15: `start button`, `reading 1`, and `remote control button` are intentional stubs; trig supports degree/radian; C Code targets real sandboxed execution.
6. **Migration notes** - document how existing IR v1 programs remain loadable while IR v2 is introduced.
